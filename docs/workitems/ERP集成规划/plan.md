## 一、背景与目标

### 1.1 业务背景

当前 Hermes Work Assistant 已完成 Phase 1-3，实现本地化文件自动化助手。现需扩展 ERP（金蝶云星空）集成能力，将系统升级为"智能办公助手"，实现：

- **零代码操作**：用户通过自然语言完成 ERP 查询、建单、附件上传
- **跨系统闭环**：微信/企微 → Agent → ERP → 微信/企微
- **数据安全**：凭证隔离、操作审计、防误操作

### 1.2 目标范围

| 目标 | 说明 | 优先级 |
|------|------|--------|
| **MCP Server 开发** | 将 kingdee_webapi_sdk 封装为 MCP Server | P0 |
| **Hermes Agent 集成** | 注册 MCP Server 为 External Server | P0 |
| **微信/企微网关** | 接入企业微信，实现 IM 交互 | P1 |
| **Skills 沉淀** | 固化常用 ERP 操作为标准化技能 | P2 |

### 3.3 安全设计

#### 3.3.1 凭证隔离

```bash
# MCP Server .env（绝不暴露给 LLM）
KINGDEE_SERVER_URL=http://your-server/K3Cloud
KINGDEE_ACCT_ID=your_acct_id
KINGDEE_APP_ID=your_app_id
KINGDEE_APP_SECRET=your_app_secret
```

#### 3.3.2 Grounding 机制

```
用户请求: "给张总建一个销售订单"
    │
    ▼
Agent 调用 query_erp_data(form_id="BD_CUSTOMER", filter_string="FName like '%张总%'")
    │
    ▼
返回: [{"FNumber": "C001", "FName": "张总贸易公司"}]
    │
    ▼
Agent 确认: "找到客户'张总贸易公司'(C001)，是否创建销售订单？"
    │
    ▼
用户确认: "是"
    │
    ▼
Agent 调用 create_erp_bill(form_id="SAL_ORDER", bill_data={...})
```

#### 3.3.3 分页截断

```python
# MCP Server 强制分页
MAX_LIMIT = 1000  # 单次查询最大返回行数

def query_erp_data(..., limit: int = 100):
    if limit > MAX_LIMIT:
        limit = MAX_LIMIT  # 强制截断
    # ...
```

## 四、风险点与避坑说明

### 4.1 高风险项

| 风险 | 影响 | 缓解措施 | 验证方式 |
|------|------|----------|----------|
| **MCP 协议兼容性** | Agent 无法发现/调用工具 | Phase 1 前用 Inspector 验证 | MCP Inspector 测试 |
| **ERP 凭证泄露** | 安全事故 | 凭证封死在 MCP Server .env | 代码审计 |
| **大数据 Token 爆炸** | LLM Context 溢出 | 强制分页（limit=100） | 压力测试 |

### 4.2 中风险项

| 风险 | 影响 | 缓解措施 | 验证方式 |
|------|------|----------|----------|
| **ERP 误操作** | 业务数据错误 | Grounding + 二次确认 | UAT |
| **微信网关不稳定** | 服务中断 | 备选 HermesClaw | 故障演练 |
| **Prompt 幻觉** | 参数错误 | Prompt 调优 + 模板 | 回归测试 |

### 4.3 避坑指南

1. **不要**将 ERP 凭证传给 LLM
2. **不要**跳过 Grounding 直接执行写操作
3. **不要**返回超过 1000 行数据给 LLM
4. **必须**在 Phase 1 前验证 MCP 兼容性
5. **必须**补充 MCP 调用链路自动化测试
