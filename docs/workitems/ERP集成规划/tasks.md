## 任务编排原则

1. **依赖优先**：T-01 (MCP 验证) 必须最先完成，否则后续任务有技术风险
2. **核心先行**：P0 任务全部完成后才能进入 Phase 3（微信接入）
3. **测试同步**：每个开发任务同步补充对应测试，不做技术债务
4. **安全第一**：T-06 (安全配置) 和 T-12 (二次确认) 必须在 Phase 2 结束前完成
5. **文档跟进**：T-21 (运维手册) 在 Phase 3 完成后立即编写
